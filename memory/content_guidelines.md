# Content Guidelines

## Strong Directions

- `Codex + 个人真实体验`
- `纯小白 + 技术学习`
- `AI 工具 + 具体工作流改变`
- `我用了多久 + 发生了什么变化`
- `具体任务 + AI 介入前后对比`
- `AI 没有让我绕开 X，反而让我意识到要学习 X`
- `工具体验 + 个人认知反转 + 可收藏学习清单`

## Weaker Directions

- 泛 AI 趋势判断。
- 只蹭热点但没有个人落点。
- 只有观点，没有过程。
- 只有工具介绍，没有具体使用场景。

## Title Rules

Good titles usually include at least two of these:

- 明确身份：纯小白、内容创作者、运营、非程序员。
- 明确对象：Codex、Python、前端、自动化、AI 工作流。
- 时间或过程：三个月后、第一次、复盘、从 A 到 B。
- 变化或反差：开始回去学、终于搞懂、把手动流程改掉。
- 具体问题：一个任务、一个工作问题、一个卡住的点。
- 认知反转：原来不是不用学，而是更知道该学什么。

Reusable formulas:

- `用 X 多久后，我把 Y 变成了 Z`
- `纯小白用 X 复盘：最有用的是 Y`
- `我和 A 一起用 X，结果 Y`
- `X 真的能解决 Y 吗？我实测了`
- `从 A 到 B，我是怎么被 X 改变的`

## Cover Rules

- 第一眼只讲一个核心信息。
- 封面和标题必须承诺同一件事。
- 优先突出结果、变化、冲突或身份。
- 少堆词，减少信息噪音。
- 系列内容保持统一视觉识别。

## Copy Rules

- 开头先给真实场景或变化，不要从大趋势开讲。
- 中段写清楚任务、过程、卡点和工具具体怎么帮上忙。
- 结尾沉淀复盘、方法或下一步，而不是空泛号召。
- 可以保留“我还在学”“我之前也不会”的普通人视角。
- 高权重结构：个人反差开头 -> 身份背景 -> 具体工具体验 -> 卡点/对比 -> 认知变化 -> 可收藏清单。
- 当主题偏教程时，要加入自己的使用过程和学习动机，避免变成纯工具说明书。

## Validated From Own Data

- `用 Codex 三个月后，我开始回去学 Python` 是当前最高权重自有样本，贡献了 60.3% 观看、85.1% 点赞、87.3% 收藏和 72.2% 涨粉。
- 该样本验证了 `Codex + 纯小白 + Python/技术学习 + 认知反转` 的组合。
- 该样本的可复用结构是：AI 工具没有替代学习，反而暴露出需要补的基础能力。
- 后续教程类内容应服务于这个主线：从真实使用中发现问题，再给出小白可执行的学习路径。

## Experiments To Run

- 同一主题测试“结果型标题”与“过程型标题”。
- Codex 系列里固定一个封面结构，观察 CTR 是否更稳定。
- 每批内容至少保留一条具体案例笔记，少量测试趋势/观点笔记。
