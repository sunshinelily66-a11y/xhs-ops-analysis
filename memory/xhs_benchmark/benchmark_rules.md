# Xiaohongshu Benchmark Rules

## Role

This folder is the same-platform benchmark layer. It should answer:

- What title patterns work on Xiaohongshu for similar topics?
- What cover structures are easy to understand at a glance?
- What body structures create saves, comments, or follows?
- Which external AI-tool topics can be translated into Xiaohongshu language?
- Which hot technical topics are too expert-heavy for this account?

## Relationship To Other Memory

- `memory/market/`: upstream trend radar. It discovers what people are discussing outside Xiaohongshu.
- `memory/xhs_benchmark/`: Xiaohongshu packaging benchmark. It validates how topics should be expressed on Xiaohongshu.
- `memory/posts/`: the user's own content evidence. It is the strongest source for whether a pattern works for this account.
- `memory/content_guidelines.md`: only stable, repeatable rules should be promoted here.

## Current Benchmark Questions

- Can a technical AI-agent topic be framed through pure beginner experience?
- Does the topic have a clear before/after workflow change?
- Can the title include identity, time/process, and concrete change?
- Can the cover explain one result or conflict in one glance?
- Does the note offer save value, not just a hot opinion?

## Avoid

- Treating Twitter/X, HN, Reddit, or GitHub heat as Xiaohongshu proof.
- Copying another creator's exact title, cover, or persona.
- Using expert-heavy framing that conflicts with the account's beginner perspective.
- Promoting a one-off benchmark sample into a long-term rule.
