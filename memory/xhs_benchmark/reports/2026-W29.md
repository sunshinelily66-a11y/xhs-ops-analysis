# 小红书 Benchmark 周报 - 2026-W29

## 样本概览
- **样本数**：5
- **主要主题**：Claude Code、AI Agent、Cursor 等 AI 工具的**教程向/安装向**内容。
- **证据质量**：5个样本均为 `low_to_medium`（教程型参考）。**关键缺失**：无封面截图、无互动数据、无评论区信号。这些样本只能用于分析“标题包装”和“工具承诺”，**不能作为“爆款模式”的证据**。

## 本周可复用的平台模式

1. **模式：教程型低门槛承诺**
   - **证据**：本周所有5个样本的标题均采用了此模式。样本中可提取的关键词包括：
     - `保姆级教程！彻底搞懂...`
     - `Windows版 X 安装与使用教程`
     - `X 分钟！！零基础彻底学会 X`
     - `15 分钟搭建你的第一个 X`
     - `X 社团实操案例：用 X 搭建工作台，自动同步 X`
   - **适合你账号的原因**：你的定位是“纯小白+AI工具实践者”，制作“用 Codex/Claude Code 完成X任务”的教程，是承接你爆款笔记 `用 Codex 三个月后，我开始回去学 Python` 的自然延伸。但需要**强制改造**：将纯功能教程，包装成**带有“我使用后的认知反转”的个人复盘教程**。

## 标题/封面/正文启发
- **标题**：
  - **可借鉴**：`时间承诺 + 零基础 + 具体工具 + 具体产出` 的结构（如`15分钟搭建AI工作台`）。这能有效降低阅读门槛。
  - **需修改**：样本标题均为纯“教程/说明书”口吻，**缺少你爆款中的个人身份和变化反差**。应修改为：`花了几天搞懂X，原来真正有用的就3步` 或 `作为小白，我用X搭了个工作台，结果Y了`。
- **封面**：
  - **结论**：样本中**不包含封面信息**，无法分析。
  - **基于账号认知的推测**：对于教程类内容，封面应避免复杂流程图。建议使用：**“一个清晰的工具界面截图”+“一个醒目的数字/结果”**（如: “3步”、“同步飞书成功”），或**“前后对比图”**（如：手动操作 vs AI自动化）。务必只讲一个核心承诺。
- **正文**：
  - **结论**：样本中**不包含正文结构**，无法分析。
  - **基于账号认知的推测**：不要直接复制“步骤1->步骤2”的说明书。建议结构为：
    1.  **开头（痛点/结果）**： “上周我还在手动...，痛苦死了。今天用X跑通了，结果爽飞了。”
    2.  **中段（我的过程+卡点）**：“虽然成功了，但中途卡在Y特别久，发现是因为我不懂Z。所以，我来复盘下这个‘小白友好版’的实操步骤...”
    3.  **结尾（清单+认知）**：“总结一下我的‘不要踩坑清单’。这次经历让我觉得，AI没有让我绕开基础，反而让我更清楚该学什么。”

## 下周可测试内容方向
- **[强推荐] 选题：用 Codex/Claude Code 做 X 工具 + 小白复盘**
  - **理由**：这是一个能在“纯教程”和“个人认知反转”之间找到平衡的选题。它既有实用价值（收藏），又有个人故事（点赞/关注）。
  - **具体化**：比如将样本 `AI 编程社团...自动同步飞书并生成信息图` 改写为：`我用Claude Code搭了个监控工作台，3天后发现最该死磕的不是代码`。这样既保留了教程本质，又叠加了你账号的认知反差核心。

- **[可实验] 选题：X分钟学会Y的“我真实验证版”**
  - **理由**：这是一个“结果型标题”的测试机会，用以对比你的“过程型标题”爆款（`三个月后，我开始回去学`）。
  - **具体化**：将 `10 分钟零基础学会 Cursor` 改写为：`说Cursor 10分钟能学会是真的吗？我实测了` 或 `10分钟学会Cursor？我只用了3分钟，但多花了2小时解决1个坑`。

## 可沉淀到 benchmark patterns 的候选
- **候选1**：`教程包装 = 个人低门槛承诺 + 操作过程 + 卡点复盘`。 此模式有潜力成为你制作教程类内容的稳定框架，但需至少 **3篇同类型样本** 验证后才能提升为长期规则。
- **候选2**：`标题中的“时间+身份”承诺`（如 `X分钟`、`零基础`）。此模式有潜力成为教程类内容的标准标题元素，但需确认其互动数据（如收藏率）高于“纯观点”标题。
