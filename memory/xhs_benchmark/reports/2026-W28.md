# 小红书 Benchmark 周报 - 2026-W28

## 样本概览
- **样本数**：5条（全部为 `low_to_medium` 权重）
- **主要主题**：Claude Code 教程、AI 编程入门、AI Agent 搭建、飞书自动化
- **证据质量**：仅有标题及外部引用片段，无封面图、正文结构、互动数据、评论信号。无法判断爆款潜力或真实点击表现。

## 本周可复用的平台模式
1. **模式：低门槛教程承诺（“保姆级/零基础/X分钟/第一个作品”）**
   - **证据**：5条样本均包含此类表达，如“保姆级教程！彻底搞懂 Claude Code Skill！”、“10 分钟！！零基础彻底学会 Cursor AI 编程”、“15 分钟搭建你的第一个 AI 智能体”、“Windows版 Claude Code 安装与使用教程”、“AI 编程社团实操案例：用 Claude Code 搭建工作台”。
   - **适合你账号的原因**：账号定位为“纯小白视角 + AI 工具实践者”，低门槛教程能降低用户的心理进入成本。但需注意你的账号信任感来源是“真实使用过程＋个人复盘”，而非纯工具说明书，因此不可照搬教程口吻。

## 标题/封面/正文启发
- **标题**：先用“保姆级”、“零基础”、“X分钟”吸引注意力，再点出具体工具（Claude Code / Cursor）或任务（搭建工作台/安装使用）。建议你在此框架下加入个人身份和变化，例如“纯小白用 Claude Code 两周复盘：只花 15 分钟就搭好了第一个自动化工作流”。
- **封面**：样本缺乏封面信息，无法提炼。建议保持你已验证的系列视觉结构（如 Codex 系列封面），在教程类内容中放一个“结果截图 + 时间数字”来强化第一眼信息。
- **正文**：样本无正文结构。根据账号定位，正文应遵循“个人反差开头 → 身份背景 → 具体使用过程 → 卡点/对比 → 认知变化 → 可收藏清单”结构，教程步骤应嵌入个人经历而非单纯罗列。

## 下周可测试内容方向
- **[可实验] 选题：**“纯小白用 Claude Code 搭建第一个自动化工作流（附飞书同步）”  
  **原因**：样本中有“AI编程社团实操案例：用 Claude Code 搭建工作台，自动同步飞书并生成信息图”，该方向同时满足“具体任务 + 工具 + 工作流变化”和“飞书自动化”这个易落地的场景。可用“15分钟/保姆级”框架包装，但加入你的个人使用过程与踩坑对比，避免变成纯教程。
