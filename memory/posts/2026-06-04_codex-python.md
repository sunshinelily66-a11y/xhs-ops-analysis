## Original Copy

用 Codex 三个月后，我开始回去学 Python

用了 Codex 三个月，我一个很直接的感受是：
AI 没有让我绕开编程，反而让我更想回去学 Python。

先交代一下背景：我本职是新媒体女工的，纯小白。队友是算法工程师。

一开始我会觉得，有了 AI，这种会写代码的工具是不是已经把门槛降得很低了。像我这种小白，是不是只要会提需求，就可以绕开“学代码”这件事。

但真的用下来发现，不是这样。

Codex 确实很强。
搭小工具、改流程、写脚本，很多以前我觉得“这我做不了”的事，它都能先帮我推起来。
但一旦报错、卡住、结果不对，我和老公的差别就会特别明显。

我看着一堆红字会发懵，
他看一眼，基本就知道问题大概出在哪。

这件事让我慢慢意识到：
编程本质上还是一门和计算机对话的语言。
AI 只是把这门语言包装得更友好了，但不代表你可以完全不懂它。

不懂代码的人，像在用一个黑盒工具，懂代码的人，更像在和一个系统协作。

现在选择从 Python 开始，只是因为它相对简单，比较适合我这种没有任何计算机基础的。

我的学习目标，也是为了先解决几个很现实的问题：

- AI 给我一段代码时，我至少能看懂它大概在干嘛
- 遇到报错时，我不至于完全不知道怎么问
- 我能自己改一点路径、变量、参数
- 我和 Codex 协作时，不会永远停留在“复制粘贴 + 祈祷跑通”

也分享一下队友给的学习建议：

1. 变量：至少知道数据现在放在哪。
2. 条件判断：看懂程序为什么走这条路，而不是另一条路。
3. 函数：知道一段代码到底在完成什么动作。
4. 列表和字典：这是 AI 给代码时最常出现的结构，看懂它，很多代码就没那么像天书了。

我现在会觉得，AI 时代，学代码其实是补上一点最基本的和计算机沟通的能力。
有点像学英语。语法不是目的，但你得先懂一点语法，才能真正交流。
